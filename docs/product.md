# 回音笺产品文档初稿

## 产品定位

回音笺是一款面向个人成长与成果显化场景的 AI 反馈系统。它通过理解用户对近期行动的自然语言描述，将原本分散、模糊、难以表达的经历，转译为结构化的可见回报，帮助用户更清楚地认识自己的投入、产出、能力增量和履历价值。

它不以任务管理为核心，也不以效率追踪为主要目标，而是聚焦“做过的事情如何被看见、被理解、被表达、被积累”这一问题。

## 用户问题

用户在日常学习、工作、求职、转岗或自我成长过程中，常常面临以下问题：

- 做了不少事情，但难以总结为具体成果。
- 明明有成长，却说不清提升了哪些能力。
- 经历很多，真正能写进简历或作品集的内容很少。
- 日常努力缺少明确反馈，长期容易失去动力。
- 很难判断当前投入是否正在形成“可见回报”。

这些问题的本质，不是用户没有行动，而是行动没有被转化为可表达、可复用、可积累的成果资产。

## 核心价值

回音笺希望提供以下核心价值：

- 把模糊叙述转成清晰结构，让用户看见自己已经完成了什么。
- 把过程中的隐性成长提炼出来，帮助用户识别能力增量。
- 把零散经历转化为可写入简历、面试或自我介绍的表达素材。
- 用“回报评分 + 下一步建议”的方式帮助用户形成持续正反馈。
- 让用户逐步建立“投入会沉淀为资产”的认知，而不是只看到未完成部分。

## 典型使用流程

### 1. 用户输入

用户输入一段自然语言，可以是文字，也可以是后续扩展的语音转写文本，例如：

“我这周整理了用户访谈记录，写了一个初版总结，还和同事讨论了后面的功能方向。”

### 2. 系统理解

系统识别这段输入中的关键内容，包括：

- 实际做了哪些动作
- 是否形成了可交付产出
- 体现了哪些能力
- 是否存在可以外显的证据
- 哪些信息适合转为简历表达

### 3. 系统输出

系统返回一份结构化结果，例如：

- 做了什么
- 产出了什么
- 提升了哪些能力
- 可以写进简历的内容
- 当前事项的回报评分
- 如何让回报更可见的下一步建议

### 4. 用户获得反馈

用户不只是“记录了做过的事”，而是得到了对成长与成果的解释，从而更容易产生持续行动的动力。

## 输出内容结构

第一版输出建议采用统一结构，便于后端处理和前端展示：

- `task_summary`：对当前输入的整体概括
- `tasks`：用户实际做过的关键动作
- `outputs`：已经形成或接近形成的产出
- `skills`：从行动中体现出的能力与成长点
- `evidence`：可以作为事实支撑的证据线索
- `resume_bullet`：适合进入简历或项目介绍的一句话表达
- `reward_score`：当前事项的可见回报评分
- `reward_reason`：评分理由
- `next_actions`：下一步如何让回报更清晰、更可展示

## 产品原则

### 不夸大

系统只能基于用户输入和合理推断进行总结，不应把未发生的结果写成既成事实。

### 不虚构

当缺少明确证据时，系统可以保守表达“可能体现了某项能力”，但不能捏造产出、数据或成果。

### 强调可见回报

输出重点不是抽象鼓励，而是帮助用户看见行动如何对应到成果、能力和履历资产。

### 帮助形成正反馈

反馈应让用户更容易感受到进展，而不是制造额外压力。系统要强调“已有价值”与“下一步可强化动作”。

### 结构清晰

输出需要稳定、简洁、可复用，便于前端展示、后续存储和模型迭代。
